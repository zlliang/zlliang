# Blog Authoring

Use this skill for content work on `tech` and `days`.

The default authoring flow is:

1. Write the draft
2. Review wording, structure, and style
3. Revise the draft
4. Tag notes only after the content is stable

## Read first

- Read [references/workflow.md](./references/workflow.md) for note and post operations
- Read [references/content-schema.md](./references/content-schema.md) before editing frontmatter or publishing content
- Read [references/style-guide.md](./references/style-guide.md) when adjusting formatting, category-specific content shape, or Markdown structure

## Review mode

- Use this skill to review blog drafts for structure, schema, formatting, and publishing readiness
- Load `english-review` only when the draft is in English and the task focuses on prose quality, tone, phrasing, or voice
- Keep content review separate from tagging; load `tag-governance` after the draft is revised and stable
- Respond in the draft's language unless the user asks otherwise
