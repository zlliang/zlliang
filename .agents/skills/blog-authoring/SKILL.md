# Blog Authoring

Use this skill for content work on `mesh` and `muse`. It is the repo-specific coordination layer for note and post workflow. It does not cover `home` fragments or copy.

## When to use

- Note and post workflow, content review, frontmatter checks, formatting checks, and publishing readiness on `mesh` or `muse`.
- The optional bilingual English-writing practice loop when the goal is practice-oriented work.

## When not to use

- As the only English prose reviewer; load `english-review` when language quality or voice matters.
- For final authorial decisions; the author owns the rewrite.

## Read first

- [references/workflow.md](./references/workflow.md) for the default note and post sequence.
- [references/content-schema.md](./references/content-schema.md) before touching frontmatter or moving content.
- [references/style-guide.md](./references/style-guide.md) for formatting, note shape, and the update note convention.
- [references/english-writing-practice.md](./references/english-writing-practice.md) only when the task is English-writing practice or bilingual comparison.

## Responsibilities

- Review drafts for structure, schema, formatting, and publishing readiness.
- Keep the distinction between author work and AI-assisted work explicit.
- Treat bilingual drafting and AI comparison as optional training loops, not mandatory steps.
- Keep the workflow lightweight and easy to follow.

## Coordination with other skills

- Pair with `english-review` for English drafts, comparison, and synthesis.
- Leave code and build changes to `website-development`.
