# Common Issues

Watch for these recurring patterns in English technical writing.

## Grammar and idiom

- Missing articles: `I wrote code` -> `I wrote the code` or `I wrote some code`
- Wrong prepositions: `focus to` -> `focus on`, `different with` -> `different from`
- Literal phrasing that follows Chinese syntax too closely
- Overused passive voice when the actor is known
- Redundant connectors or openers such as `in order to` or `the reason is because`

## Style and clarity

- Prefer direct verbs over noun-heavy phrases: `make a decision` -> `decide`
- Cut filler and repeated qualifiers unless they are deliberate for voice
- Use one term consistently after introducing it
- Keep code terms, file paths, and commands in backticks
- Keep code snippets short and explain why they matter
