# Common Issues

Watch for these recurring patterns in English technical writing.

## Grammar and idiom

- Missing articles: `I wrote code` -> `I wrote the code` or `I wrote some code`
- Wrong prepositions: `focus to` -> `focus on`, `different with` -> `different from`
- Overused passive voice: `The function was called` -> `We called the function`
- Awkward transitions: vary connectors instead of leaning on `So` or `Then`
- Redundancy: `in order to` -> `to`, `the reason is because` -> `because`

## Style and clarity

- Prefer direct verbs over noun-heavy phrases: `make a decision` -> `decide`
- Remove filler that weakens technical writing unless it is deliberate for voice
- Keep code terms, file paths, and commands in backticks
- Introduce unfamiliar terms on first use, then use them consistently
- Keep code snippets minimal and explain why they matter
