# Authoring Blogs

The note and post lifecycle on `mesh` and `muse`: creating, drafting, frontmatter and schema, formatting, update notes, and publishing readiness. Does not cover `home`.

## Routing between skills

- Load `reviewing-english` alongside this skill whenever an English draft's language quality or voice matters; this skill coordinates the workflow but does not review prose.
- Leave code, build, and `toolkit` behavior changes to `developing-websites`.
- The author owns drafting and final rewrites; AI reviews, structures, and implements, but does not replace the author's thinking.

## Read first

- [references/workflow.md](./references/workflow.md) — the default note/post sequence; start here.
- [references/content-schema.md](./references/content-schema.md) — before touching frontmatter or moving content files.
- [references/style-guide.md](./references/style-guide.md) — titles, italics, quotation marks, update notes, and dates.
- [references/english-writing-practice.md](./references/english-writing-practice.md) — only when the task is English-writing practice or bilingual comparison; this loop is optional, not part of normal publishing.
